# CBID Workshop: 5 Practical Exercises

## Exercise 1: Timing Pattern Recognition

**Objective**: Develop skill in identifying timing patterns from constraint observations.

**Scenario**: A web service shows intermittent 5-second delays every 30 minutes.

**Tasks**:
1. Document the constraint activation time
2. Identify the pattern type (threshold, interval, split, velocity)
3. Measure exact timing between delays
4. Note any variations in the pattern
5. Develop hypothesis about what causes the pattern

**Learning outcomes**:
- Practice timing measurement and documentation
- Learn to categorize pattern types
- Develop hypothesis from observable patterns

---

## Exercise 2: False Alarm Analysis

**Objective**: Learn systematic verification to avoid false positives.

**Scenario**: At exactly 60 minutes into a constraint, you detect what appears to be resolution.

**Tasks**:
1. Document the apparent resolution signature
2. Design verification checks across multiple domains
3. Identify potential false positive indicators
4. Create verification protocol for future similar situations
5. Document what you learn from the analysis

**Learning outcomes**:
- Understand psychological threshold effects
- Develop systematic verification approach
- Learn from false positive experiences

---

## Exercise 3: Velocity Mismatch Calculation

**Objective**: Quantify speed differences between system domains.

**Scenario**: Physical package tracking shows 2-hour delay, while digital notification system updates every 15 minutes.

**Tasks**:
1. Document timing in physical domain (2 hours = 120 minutes)
2. Document timing in digital domain (15 minutes)
3. Calculate velocity ratio (120:15 = 8:1)
4. Identify what the ratio reveals about system architecture
5. Predict how changes in one domain affect the other

**Learning outcomes**:
- Practice velocity calculation
- Understand domain speed differences
- Learn to interpret velocity ratios

---

## Exercise 4: Specialization Pattern Identification

**Objective**: Recognize how constraints drive role emergence.

**Scenario**: Team facing a production issue with multiple team members responding differently.

**Tasks**:
1. Document different team member responses to the constraint
2. Identify specialization patterns emerging
3. Map how constraints create different response niches
4. Analyze coordination patterns between specialized roles
5. Develop insights about team self-organization

**Learning outcomes**:
- Recognize constraint-driven specialization
- Understand niche formation in systems
- Analyze multi-agent coordination patterns

---

## Exercise 5: Hypothesis Development & Testing

**Objective**: Create and test predictions from constraint patterns.

**Scenario**: Database queries slow down predictably at 2 PM daily.

**Tasks**:
1. Document the timing pattern (daily 2 PM slowdown)
2. Develop testable hypothesis about cause
3. Design verification experiment
4. Make specific prediction about tomorrow's behavior
5. Document verification results and learning

**Learning outcomes**:
- Practice hypothesis development from patterns
- Design verification experiments
- Learn from prediction outcomes
- Refine understanding based on results

---

## Workshop Facilitation Guide

**Time allocation**: 60-90 minutes total
- Introduction: 10 minutes
- Exercise 1: 15 minutes
- Exercise 2: 15 minutes  
- Exercise 3: 10 minutes
- Exercise 4: 10 minutes
- Exercise 5: 10 minutes
- Wrap-up: 10 minutes

**Materials needed**:
- Timer/stopwatch
- Note-taking tools
- Example constraint scenarios
- CBID Cheat Sheet reference

**Success criteria**:
- Participants can identify at least 3 pattern types
- Participants develop testable hypotheses
- Participants understand velocity ratio calculation
- Participants recognize specialization patterns
